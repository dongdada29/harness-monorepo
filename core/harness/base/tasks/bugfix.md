# Bugfix Template - Bug 修复模板

> 适用于所有项目  
> **版本**: 2.0.0（新增决策启发 + 规范约束）

---

## 结构总览

```
完整 Skill = 操作过程 + 决策启发 + 规范约束

操作过程 → 第一节（做什么）
决策启发 → 第二节（怎么做分支判断）
规范约束 → 第三节（做到什么程度算合格）
```

### CP1: 规划

- [ ] 确认 Bug 信息填写完整
- [ ] 根因分析至少有一条确定的代码位置
- [ ] 修复方案描述清楚改动范围
- [ ] **Human approve** 后进入 CP2

### CP2: 执行

- [ ] 按修复方案修改代码
- [ ] 修改文件不超过 5 个（`constraints.md` 规定）
- [ ] 每次文件修改后更新 `state.json` 的 checkpoint
- [ ] 遵守日志规范（参考 `logging.md`）

### CP3: 验证

- [ ] lint 通过
- [ ] typecheck 通过（如果有）
- [ ] 测试通过
- [ ] 手动复现确认 Bug 不再出现
- [ ] 无回归（跑全量测试或关键路径测试）

### CP4: 完成

- [ ] 更新 `state.json` 的 `tasksCompleted` +1
- [ ] 如果是新 Bug 类型，追加到 `patterns` 字段供以后检索

## 第二节：决策启发 (Decision Heuristics)

> 遇到以下情况时，按照对应分支处理

### 编译错误

```
遇到编译错误 →
  ├─ 类型错误 → 先查 .d.ts 文件是否缺失或版本不对
  ├─ 语法错误 → 直接看报错行号，大多数是打字错误
  └─ 链接错误 → 查是否缺少导出或循环依赖
```

### 测试失败

```
测试失败 →
  ├─ 新代码引入的失败 → 修复新代码，测试是金标准
  ├─ 老代码原本就失败 → 标记为已知问题，不阻塞修复
  └─ 偶发性失败 → 重跑 3 次，3 次都失败才算真失败
```

### 遇到阻塞

```
遇到阻塞（30 分钟内无进展）→
  ├─ 查 Memory（CP2 EXEC 检索）→ 有相似案例则复用
  ├─ 无相似案例 → 升级到 CP3，人工介入
  └─ 连续 3 次同类阻塞 → 触发 autonomy.md L3 人工介入
```

### 修复范围蔓延

```
修复过程中发现额外问题 →
  ├─ 同模块的小问题 → 一起修，记录在修复描述里
  ├─ 不同模块的问题 → 记录在 state.json 的 `pendingTasks`，不开新分支
  └─ 超出原问题范围 → 停止，汇报，等人类确认
```

### 修复方案不可行

```
修复后发现方案有漏洞 →
  ├─ 立即回滚（git revert 或手动回退）
  ├─ 记录失败原因到 state.json
  └─ 重新进入 CP1，重新规划
```

## 第三节：规范约束 (Normative Constraints)

> **Gate 定义**：以下条件必须全部满足，CP3 才能标记为 completed

### 必须满足

| 约束 | 说明 |
|------|------|
| **lint 通过** | `npm run lint` 或项目对应 lint 命令，exit code = 0 |
| **测试通过** | 相关单元测试全部 pass，无新警告 |
| **Bug 不复现** | 手动复现步骤走一遍，确认问题消失 |
| **无回归** | 全量测试或关键路径测试通过 |
| **文件修改 ≤ 5 个** | 超过 5 个文件需要额外说明理由 |
| **每次 checkpoint 更新 state.json** | CP2 执行过程中每修改一个文件，checkpoint 记录一次 |

### 不允许

| 约束 | 说明 |
|------|------|
| **不允许热修复** | 禁止直接改生产代码，必须走完整流程 |
| **不允许跳过验证** | 即使很自信，也必须跑测试 |
| **不允许强制提交** | `git push -f` 禁止在修复分支上使用 |
| **不允许删除测试** | 测试是规范约束的一部分，删测试 = 不合格 |

### 合格标准

```
CP3 Verify 通过当且仅当：
  lint === pass AND
  test === pass AND
  bug_not_reproducible === true AND
  no_regression === true AND
  file_count <= 5
```
