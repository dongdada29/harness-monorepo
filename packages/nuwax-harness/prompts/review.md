# 代码审查模板

## 自审清单

### 1. 逻辑正确性
- [ ] 代码实现了需求功能
- [ ] 边界条件处理了
- [ ] 错误处理完善

### 2. 代码质量
- [ ] 无重复代码
- [ ] 函数 < 50 行
- [ ] 变量命名清晰
- [ ] 无硬编码值

### 3. 测试覆盖
- [ ] 有对应测试
- [ ] 边界条件有测试
- [ ] 测试可读可维护

### 4. 提交规范
- [ ] 无 console.log
- [ ] 无敏感信息
- [ ] commit message 清晰

---

## 问题严重程度

| 等级 | 说明 | 处理方式 |
|------|------|----------|
| 🔴 严重 | 必须修复，否则功能不能用 | 立即修复 |
| 🟡 中等 | 应该修复，但不影响功能 | 尽快修复 |
| 🟢 轻微 | 可选修复 | 可以下个迭代 |

---

## 输出格式

```
🔍 Review: [文件]

✅ 通过:
- [检查项 A]
- [检查项 B]

❌ 问题:

🔴 严重:
- [文件:行号] [问题描述]

🟡 中等:
- [文件:行号] [问题描述]

⚠️ 建议:
- [建议描述]

📊 总结:
- 严重: N
- 中等: N
- 轻微: N
```
