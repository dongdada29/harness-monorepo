# Harness 效果评估体系

> 如何判断 Harness 是否有效

---

## 评估维度

### 1. 效率指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **任务完成率** | Agent 完成的任务 / 总任务数 | `completed / total` |
| **平均任务时长** | 从 /start 到 /done 的平均时间 | `sum(duration) / count` |
| **Gate 通过率** | 首次 /verify 通过的比率 | `first_try_pass / total` |
| **阻塞率** | 被阻塞的任务比率 | `blocked / total` |
| **上下文使用率** | 平均上下文消耗 | `avg(context_used / max)` |

### 2. 质量指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **Lint 错误数** | 每次 verify 的 lint 错误 | `sum(lint_errors)` |
| **Type 错误数** | 每次 verify 的类型错误 | `sum(type_errors)` |
| **测试覆盖率** | 平均测试覆盖率 | `avg(coverage)` |
| **Bug 回归率** | 修复后再次出现的问题比率 | `regression / fixed` |

### 3. 行为指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **约束违规率** | 违反约束的次数 | `violations / total` |
| **阻塞汇报率** | 正确使用 /blocked 的比率 | `blocked_reports / blocks` |
| **状态更新率** | 按时更新 state.json 的比率 | `updated_on_time / total` |

### 4. 自主能力指标

| 指标 | 说明 | 计算方式 |
|------|------|----------|
| **人类介入率** | 需要人类决策的比率 | `human_decisions / total` |
| **自主完成率** | Agent 独立完成的任务比率 | `solo_completed / total` |

## 评估方法

### 方法 1: A/B 测试

对比使用 Harness 前后的差异：

```
使用前 30 天 vs 使用后 30 天

| 指标         | 使用前 | 使用后 | 变化 |
|--------------|--------|--------|------|
| 任务完成率   | 65%    | 82%    | +17% |
| 平均时长     | 45min  | 32min  | -29% |
| Gate 通过率  | 70%    | 91%    | +21% |
| 阻塞率       | 25%    | 8%     | -17% |
```

### 方法 2: Gate 失败分析

分析 Gate 失败的原因分布：

```
Gate 1 (Lint):  35% - 代码风格问题
Gate 2 (Type):  20% - 类型定义问题
Gate 3 (Test):  30% - 测试覆盖不足
Gate 4 (Build): 15% - 构建配置问题
```

### 方法 3: 阻塞原因分析

分析 /blocked 的原因分布：

```
技术难题:     45% - 需要人类决策
缺少上下文:   30% - Harness 上下文不够
工具限制:     15% - 权限/工具不足
资源依赖:     10% - 等待外部资源
```

## 评估流程

```
┌─────────────────────────────────────────────────────────────┐
│  Step 1: 收集数据 (每周)                                   │
│  - 从 state.json 统计任务完成/阻塞                          │
│  - 从 CI 日志提取 Gate 失败率                             │
│  - 从 git log 分析 commit 质量                            │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 2: 计算指标 (每周)                                   │
│  - 效率: 任务完成率、平均时长、Gate 通过率                  │
│  - 质量: lint/type 错误数、覆盖率                          │
│  - 行为: 违规率、阻塞率、状态更新率                        │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 3: 生成报告 (每周)                                   │
│  - 与上周/上月对比                                          │
│  - 识别异常指标                                            │
│  - 提出改进建议                                            │
└─────────────────────────┬─────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│  Step 4: 调整 Harness (每月)                               │
│  - 根据数据调整约束                                        │
│  - 补充缺失的上下文                                        │
│  - 优化工作流                                             │
└─────────────────────────────────────────────────────────────┘
```
